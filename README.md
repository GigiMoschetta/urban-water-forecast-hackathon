*"From Bills to Plans: Predicting 12 Months of Water Demand at Urban Scale"*

| | |
|---|---|
| **Task** | Forecast 12 months of monthly water consumption (m3) per spatial cell and tariff category |
| **Primary model** | `ensemble_3m_v1_v9o_nested` — 3-model ensemble + nested CV correction |

> ⚠️ **About the data in this public repo.** The hackathon dataset is confidential and cannot be redistributed. This repository ships a deterministic **synthetic placeholder** (~200 m³ constant + tiny noise) generated by `scripts/generate_synthetic_data.py`, so the demo dashboard and the build tooling run end-to-end out of the box without any real data. The performance numbers above refer to evaluations on the original confidential dataset, not on the synthetic placeholder.

## Quick start

### With Docker or Podman (any OS)

```bash
git clone <this-repo-url> && cd urban-water-forecast-hackathon
```

**Launch the demo dashboard** (uses the synthetic placeholder shipped in the repo):

```bash
make demo
# then open http://localhost:3000
```

### Without Docker — run the demo

```bash
pip install -r requirements.txt
PYTHONPATH=src python scripts/generate_synthetic_data.py   # writes synthetic CSVs (idempotent)
PYTHONPATH=src python demo/scripts/build_data.py           # writes demo/public/data.json
cd demo && npm install && npm run dev                      # http://localhost:3000
```

### Reproducing the model on the real dataset

`make verify` and `make reproduce` are intended for evaluators who hold a copy of the original `units_hackathon_challenge_dataset.xlsx`. Place it in `data/original/` and run:

```bash
make verify       # check pre-computed predictions match the historical metrics
make reproduce    # rebuild the 3-model ensemble (~10 min)
```

These targets will not produce the headline metrics on the synthetic placeholder — they need the real dataset.

## Project overview

Urban water demand forecasting at the 12-month horizon for AcegasApsAmga, serving **Padova (PD)** and **Trieste (TS)**. The dataset covers 163 spatial cells (H3 resolution 6 and 7), 5 tariff categories, and 72 months (2020-01 to 2025-12). Data is anonymized with a multiplicative noise model where the shared monthly factor cancels in cross-cell comparisons — making spatial features higher quality than temporal ones.

The model is framed as a **planning-support system**: cell-level forecasts for network sizing, staffing, anomaly review, and risk triage.

## Modeling approach

**Why global boosted trees**: 72 data points per series — too short for local models. A single global LightGBM learns shared patterns across all series while adapting via meta-features. All targets in `log1p` space for variance stabilization.

**Direct multi-horizon**: all 12 months predicted in one pass with `horizon` as input feature. No error propagation, better ensemble composability vs recursive.

**Feature layers** (see `features.py`):
1. **Temporal**: lags (1,2,3,6,12,24), rolling means/stds, EWM, YoY delta/ratio
2. **Spatial**: neighbor mean, spatial ratio, node degree — exploits shared anonymization factor cancellation
3. **Meta**: series mean volume, CV, trend strength, ACF12 (per-fold, leakage-safe)
4. **Domain (V9)**: gamma-t proxy, March anchor, regime flags (COVID/drought/smart meters), territory interactions, volume density

### Primary model: 3-stream ensemble

Each model serves a distinct, non-redundant role (validated by ablation):

| Stream | Type | Tuned MAPE | Role in ensemble |
|--------|------|----------:|-----------------|
| `naive` | Seasonal naive | 27.91 | Per-series fallback for 38 volatile series |
| `v1` | Recursive HistGBR | 23.57 | Dominates short horizons h=1-4 (weight ~0.64) |
| `v9_optuna` | Domain-SOTA LightGBM | 22.16 | Dominates long horizons h=5-12 (weight ~0.75) |

v3, v3d, v8 were removed after ablation showed they are redundant with v9 (+0.04 MAPE, simpler model).

**Ensemble pipeline**:
1. Nelder-Mead weight optimization per horizon band (global or per-resolution, best wins)
2. Per-series naive fallback for volatile series (38 series where naive beats the blend)
3. Nested CV residual correction (7 params, leave-one-fold-out, Optuna)

## Experimentation workflow

To test a new model combination:

```bash
# Try any subset of: naive, v1, v3, v3d, v8, v9, v9o, v9w, ridge
PYTHONPATH=src python scripts/run_ensemble_ablation.py \
  --models naive,v1,v9o \
  --output-name my_experiment \
  --trials 100
```

To promote a new best model, update `scripts/run_ensemble_compact5.py` (change `DEFAULT_MODELS` and `DEFAULT_OUTPUT`) and `src/hackathon_opti/model_registry.py`.

## Data quality notes

- **Duplicate keys**: natural key `(cell_id, rate_category_id, period_ym)` is not unique for some H3-resolution-7 series. Handled by summing — sub-tariffs must be aggregated.
- **Anonymization**: `y' = alpha_c * gamma_t * y + epsilon`. Cross-cell ratios at same timestamp are reliable (gamma_t cancels). Absolute scale is lost.
- **Structural breaks**: COVID 2020, drought rationing 2022, smart meter rollout 2023, extreme heat 2024.

See [docs/02_dataset.md](docs/02_dataset.md) for full dataset documentation and [docs/03_model.md](docs/03_model.md) for model architecture details.

## Known limitations

1. **Fold 4 hardest** (~27% MAPE) — 2024-2025 may contain unseen structural changes
2. **Small volatile series** — Cluster 1 (6.5% of series) generates ~40% of total error
3. **No exogenous variables** — temperature unavailable; Fourier harmonics approximate but can't capture anomalies
4. **Resolution 6 harder** — 36 km2 cells, only 69 series, MAPE ~26% vs ~19% for resolution 7

## Approaches tested and discarded

| Approach | Result | Why |
|----------|--------|-----|
| Share-based models (v4, v6) | MAPE ~40% | gamma_t estimation errors amplify |
| LightGBM MAPE objective | MAPE 62% | Gradient explodes on near-zero series |
| Sample weighting by CV | +1.33 MAPE | Trees overfit noise on volatile series |
| Cluster routing to naive | Worse | Ensemble already handles volatile series |
| Resolution-specific models | +0.50 MAPE | Res 6 has only 69 series, overfits |
| STL decompose + model residual | 27.9% | Linear trend extrapolation fails |
| Multi-window training (M5) | +0.57 MAPE | 72 months too short |
